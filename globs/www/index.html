Beyond POJOs: a desktop application framework based on "Generic Lightweight OBjectS"

Introduction

Globs is an application framework in which the model part uses generic objects based on a formal metamodel description. These generic objects offer two main advantages over traditional POJOs / Java Beans:

  • Since all the model objects use the same Java class, it is very easy to build reusable code and components that work with any model entity.
  • The meta-model (definition of types, fields and constraints on these fields) is accessible through a number of plain Java APIs that are much easier to deal with than the introspection APIs. This also makes it very easy to build reusable components.

With this approach, we provide a number of generic services:

  • model-driven GUI components that keep themselves in sync with the underlying model objects
  • transparent SQL mapping
  • transparent XML mapping
  • binary serialization for client/server exchanges
  • etc.

Theoretically, this concept can be used for many kinds of applications. However, we are now strongly focused on the desktop application area.

We have implemented various aspects of this approach in large business applications, our goal now is to build a comprehensive open source framework based on these concepts. This development is guided by real-world needs, since we are using it to build a commercial application.
